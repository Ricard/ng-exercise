  • You can find a new location that lazy load the user-list mopdule: Users list

    As usual, the HTTP request for retrieving users is done by a service. The results are only retrieved one time and stored in this singleton.

  • The component Users list is navigating to user detail component.

    Taking the advantage of a service, to retrieve a user information does not requiere a new http request. The service is able to return all the users list or just only one.

    Regarding to styles, I have to explain that I didn't introduced any CSS framework but just copyed some W3 schools CSS https://www.w3schools.com/w3css/

  • The solution could be found in home section

    There are many approaches for comunication between components. Frequently it is requiered a service but in this case I decided to handle the comunication from the parent component as you suggest.

    Each component have a boolean variable that represents the login status. I decided to change the component for accepting a input, therefore parent is able to change the components login status.

    In the other hand, each component will notify its login status by a output stream. When this output have a new value, the parent store it in the same variable that is an input for the other component.
